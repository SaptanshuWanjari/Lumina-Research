# 02. Database Schema and State Models

## Overview

This specification details the structure for the `supabase` PostgreSQL database, based on the AI Research and Decision Workspace PRD. The application is single-user, and the core principle is that **every app row is owned by a single authenticated user (**`owner_user_id`**)**. This is enforced via Row Level Security (RLS) policies and explicit application-level checks.

### Required PostgreSQL Extensions

- `pgvector`: Essential for storing semantic embeddings in the `chunks` table.
- `uuid-ossp`: Essential for generating UUIDs.

---

## Entity Specifications

### 1. `profiles`

Mirrors authenticated users linked to `auth.users` in Supabase.

- `id` (uuid, primary key, references `auth.users`)
- `created_at`, `updated_at` (timestamps)
- `full_name`, `avatar_url` (text)

### 2. `cases`

The fundamental unit of research.

- `id` (uuid, primary key)
- `owner_user_id` (uuid, not null)
- `title` (text, max 255)
- `question` (text, optional)
- `status` (enum: `draft`, `ingesting`, `indexed`, `analyzing`, `review`, `published`, `failed`, `archived`)
- `priority` (text/integer)
- `tags` (jsonb/array)
- `created_at`, `updated_at`, `archived_at` (timestamps)

### 3. `sources`

Input materials (URLs, PDFs, notes) linked to a `case`.

- `id` (uuid, primary key)
- `case_id` (uuid, references `cases`)
- `owner_user_id` (uuid, not null)
- `source_type` (enum: `url`, `file`, `note`)
- `url` (text, optional)
- `storage_path` (text, optional, points to Supabase Storage bucket)
- `note_text` (text, optional)
- `status` (enum: `pending`, `fetching`, `extracting`, `chunking`, `embedding`, `indexed`, `failed`, `archived`)
- `error_message` (text, optional)
- `content_hash` (text)
- `created_at`, `updated_at` (timestamps)

### 4. `documents`

Normalized, extracted content derived from raw `sources`.

- `id` (uuid, primary key)
- `source_id` (uuid, references `sources`)
- `case_id` (uuid, references `cases`)
- `owner_user_id` (uuid, not null)
- `content` (text)
- `created_at` (timestamp)

### 5. `chunks`

Retrievable text segments linked to a `document`, embedded for vector search.

- `id` (uuid, primary key)
- `document_id` (uuid, references `documents`)
- `case_id` (uuid, references `cases`)
- `owner_user_id` (uuid, not null)
- `chunk_index` (integer)
- `content` (text)
- `token_count` (integer)
- `embedding` (vector(1536), assuming OpenAI `text-embedding-3-small` or similar)
- `metadata_json` (jsonb)

### 6. `runs`

Execution instances of the LangGraph AI workflow for a specific `case`.

- `id` (uuid, primary key)
- `case_id` (uuid, references `cases`)
- `owner_user_id` (uuid, not null)
- `status` (enum: `queued`, `running`, `needs_review`, `resuming`, `complete`, `failed`, `cancelled`)
- `current_step` (text)
- `needs_review` (boolean, default false)
- `checkpoint_ref` (text, thread_id for LangGraph checkpointer)
- `checkpoint_at` (timestamp)
- `started_at`, `completed_at` (timestamps)
- `error_message` (text, optional)
- `approved_by_user_id` (uuid, optional)
- `approved_at` (timestamp)

### 7. `run_steps` & `run_artifacts`

Node-level execution traces and structured outputs (retrieval sets, critique notes, validation) for observability and debugging of LangGraph runs.

- Must include `run_id`, `owner_user_id`, `step_name`, `status`, `duration_ms`, `input_json`, `output_json`.

### 8. `report_versions`

Versioned, human-reviewable artifacts generated by `runs`.

- `id` (uuid, primary key)
- `case_id` (uuid, references `cases`)
- `run_id` (uuid, references `runs`)
- `owner_user_id` (uuid, not null)
- `version_number` (integer)
- `status` (enum: `draft`, `published`, `archived`)
- `content_markdown` (text)
- `citations_json` (jsonb)
- `created_at`, `published_at` (timestamps)

### 9. `report_claims` & `report_citations`

Records linking claims in a `report_version` to specific `chunks` and `sources` for full traceability and explainability.

### 10. `jobs` (Optional/Queueing)

If using a DB-backed queue instead of Celery/Redis for Python workers.

- Needs `id`, `task_type`, `payload_json`, `status`, `attempts`, `error`, `created_at`, `updated_at`.
